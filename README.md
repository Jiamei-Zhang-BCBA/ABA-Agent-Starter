# ABA-Clinical-Agent-Starter 🧩

这是一个**开箱即用的自动化 ABA 临床督导知识库系统**。
它将 [Obsidian 笔记库] 与 [大语言模型 Agent] 进行了无缝结合。当你在支持本地工作区的大模型客户端（如 Cursor、Claude Desktop、Cline 等）中打开本文件夹时，你的 AI 将瞬间化身为一位具备 **BCBA 专业素养的临床总监**，帮你全自动处理从个案建档、IEP 生成、督导听课反馈一直到最终结业撰写的全套临床流转。

---

## 📂 目录结构说明

本系统由一明一暗两个核心构成：

1. **`Obsidian-Vault/` (明线：你的业务笔记库)**
   这是你平时用来记录个案档案、存放评估数据、撰写家长周报的专属空间。建议直接用 Obsidian（黑曜石）软件作为 Vault 打开此文件夹。
   - `00-RawData/`：存放脱敏前的原始素材（访谈记录、听课纸）
   - `01-Clients/`：核心区！每个儿童的终身数字档案 `Client-[代号]-核心档案.md`
   - `[...]` 02到06 包含了人事、模板和沟通记录。
2. **`.claude/skills/` 与 `CLAUDE.md` (暗线：系统的智能引擎)**
   这包含了长达几千行的 17 个核心自动化临床技能树（Skills）、行为干预与 VB-MAPP 知识字典（references），以及赋予大模型专业判断边界的基线脑。**日常不要随意修改里面的代码，除非你想给大模型升级！**

## 🚀 极其简单的快速上手 (Quick Start)

**准备要求：**
- 一个安装了 Cursor 客户端、Claude Code 或任何能挂载整个本地文件夹的智能助理。

**第一步：召唤督导系统**
将刚刚解压的根目录 `ABA-Agent-Starter` 拖入你的 Cursor (或在文件夹根目录运行大模型命令)。你会看到左侧出现了 Obsidian-Vault 文件夹。

**第二步：唤醒第一道指令**
打开大模型对话框（Cmd+L 等），向 AI 发出明确的业务指令即可。比如，你可以直接粘贴以下咒语作为系统的第一声问候：

> "请阅读根目录的 `CLAUDE.md` 并进入督导角色。我刚刚接收了一个新个案叫‘多多’。请调用 `intake-interview` 技能，并告诉我你需要我提供什么信息？"

## 🧠 这套系统能干什么？(17 项自动化能力矩阵)

AI 会严格遵循 `skills/_router.md` 的工作流，无需你死记硬背。最常见的几个威力巨大的操作：

- 👤 **建立新档案**：“请用 `intake-interview` 为 Client-龙龙 建档。” 
  *(AI 会自动在 01-Clients 生成专属文件夹及他的全套核心档案表）*
- 📝 **写出专业 IEP**：“根据兜兜刚测完的评估表，用 `plan-generator` 帮我写一份短程目标规划。”
  *(AI 会翻出档案柜里兜兜的所有旧账，推导出合理的优先级和教学策略阶梯)*
- 👩‍🏫 **听课点评与师训**：“我今天去看了张老师给涵涵上的课，发现她经常做反向辅助。用 `staff-supervision` 帮我自动记录。”
  *(AI 会在 03-Staff 自动追加这位老师被纠错的记录，并存入你的成长档案系统，永不遗落)*
- 💌 **给家长写高情商家书**：“每周一的家长更新要发了，执行 `parent-update`。”
  *(AI 会调取上一周家长回馈的所有进度，自动生成一篇带着情绪价值的精美总结反馈排版)*

### ⚠️ 关于安全与数据的绝对边界
为了防止 AI 幻觉破坏你的珍贵档案，系统已经内置了硬防线：
遇到**修改和新建**敏感文档时，大模型会默认触发预览（Diff）。在对话框里展示后，只有你亲口说“确认/好/可以执行”，它才会把文字永远写入本地文档中！

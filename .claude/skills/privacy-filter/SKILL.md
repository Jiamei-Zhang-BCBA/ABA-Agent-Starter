**极易踩坑的半脱敏写法 — 严禁使用**：
- ❌ `Client-A-小磊 母亲（王丽，38岁，某银行部门总监）` — 把代号和真名**并列**出现 = 映射被泄漏，等同于未脱敏

**正确写法**：

- ✅ 变更日志：`个案代号：Client-A-小磊；数据来源：初访家长访谈笔录；脱敏操作完成。` — 完全不提真名

# 🛑 真名变体全捕获 (P1 硬规则 — BUG #23 对应)

**问题背景**：家庭成员访谈中互相使用的"姓氏 + 职业敬称 / 亲属称谓"在 v4
测试暴露脱敏缺口——真名 `李建国 / 陈婉仪` 被正确替换，但文中的 `李老师 / 陈老师 / 李教授 / 陈教授` 未被识别为同一人的代称，8 处残留在脱敏存档。

**识别算法**（每个真名 X 都必须做以下扩展检索）：

当身份映射表中录入真名 `姓 + 名` 后，自动识别并替换该人的**所有变体**：

| 变体类型 | 示例（以 "李建国" 为例） | 处理 |
|:---|:---|:---|
| 姓氏 + 职业敬称 | 李老师、李教授、李医生、李律师、李工、李经理、李总、李书记、李主任、李先生 | 全部替换为 `[Client-代号 父亲]` 或统一角色代称 |
| 姓氏 + 职称缩写 | 李博（博士）、李副（副教授）、李总工、李局 | 同上 |
| 姓氏 + 女士 / 先生 | 李先生、陈女士、王太太 | 同上 |
| 昵称变体 | 建国、国国、小李（对外人自称） | 需配合上下文判断是否代称同一人 |
| 复合关系暗桩 | "李教授的夫人就是陈老师"、"磊磊爸就是张强"、"她家老李" | **此类暗桩必须整句删除或彻底改写**，绝不能保留"暗示链" |

**扩展检索模板**（对每个身份映射表里的真名 `Name = 姓 + 名`，脱敏引擎必须做）：

```
For name in known_names:
    surname = name[0]  # 单姓；若复姓特殊处理（欧阳/司马等）
    variants = [
        surname + "老师", surname + "教授", surname + "医生",
        surname + "律师", surname + "总", surname + "博",
        surname + "经理", surname + "主任", surname + "书记",
        surname + "处长", surname + "先生", surname + "女士",
        surname + "太太", surname + "工程师", surname + "局长",
        name[1:],  # 去姓的名字
        "小" + surname, "老" + surname,
    ]
    for v in variants:
        if v in text:
            flag_for_review(v, context=neighbor_100_chars)
            text = text.replace(v, canonical_role_reference)
```

**边界情况**：
- **非 PII 姓氏撞车**：如果文中出现"李医生"但这是**另一个**真人（非家庭成员），也应脱敏为"某医院医生"
- **保留职业无姓**：`医生`、`老师`（无姓前缀）单独出现时**不必**脱敏（职业本身非 PII）

**本规则适用场景**：第二步"执行多准则脱敏"时，必须对身份映射表里**每一个真名**执行一次"变体扩展扫描"。

# 🔍 第 3.5 步：脱敏自检清单 (预览前必做)

在第四步写入脱敏存档**之前**，必须对即将写入的内容逐项自检，打钩通过才能进入预览：

- [ ] 脱敏存档全文 grep "真名" = 0 命中（包括但不限于父母/祖辈/保姆/老师/医生姓名）
- [ ] 脱敏存档全文 grep 具体学校名/医院名/公司名 = 0 命中（只保留 "某"字类描述）
- [ ] 变更日志草稿 grep 真名 = 0 命中
- [ ] 变更日志草稿 grep 身份证/门牌号/微信号/手机号 = 0 命中

**自检未通过时的处置**：在响应中明确输出 `⚠️ 自检未通过：[具体位置]，已回到第二步重做`，不得静默绕过。

# 执行步骤与多重文件操作
请你必须严格按照以下顺序，在本地执行**脱敏与绝密归档**操作：

**第一步：绝密对照表检索 (Read & Match)**
1. **指令**：读取 `00-RawData/身份映射对照表-绝密.md`。
2. **逻辑判定**：
   - 检查输入文本中的真实姓名是否已存在。
   - **已有案例**：沿用表中对应的系统代号（如 `Client-A-兜兜`）。
   - **新增案例**：根据表中现有数量，自动分配下一个顺延代号（如 `Client-C-小明`）。

**第二步：执行多准则脱敏 (De-identify)**
1. **强制替换**：将所有真实姓名替换为分配的代号。
2. **模糊处理**：将具体地址、学校全称、父母全名等替换为描述性词汇（如"某公立小学"、"Client-A 妈妈"）。
3. **保留专业度**：严禁修改任何关于行为频次、前因后果（ABC）、干预时长等专业数据。

**第三步：更新绝密映射表 (Append - 仅限新个案)**
1. **操作指令**：如果是新出现的孩子，请将对应关系追加到映射表末尾。
   - 目标路径：`00-RawData/身份映射对照表-绝密.md`
   - 追加格式：`| 真实姓名 | 系统代号 (Aliases) | 录入日期 |`

**第四步：存入脱敏存档库 (Write)**
1. **预览确认**：将脱敏后全文输出给督导，确认无遗漏后执行写入。
2. **操作指令**：新建文件写入净化后的文本。
   - 目标路径：`00-RawData/脱敏存档/`
   - 文件名：`[[代号-脱敏原始数据]].md`
   - 写入内容：参照下方的【输出规范】。

**第五步：变更日志 (Append)**
1. **操作指令**：追加至 `04-Supervision/系统变更日志.md`。
